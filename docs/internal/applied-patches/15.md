# Patch 15 — mcp.sw:82

## Problem
`mcp_log_path` concatenates an unsanitized server name from user config directly into a filesystem path, allowing path traversal via `../` or absolute paths.

## Current code
```sw
fun mcp_log_path(server) {
    getenv("HOME") ++ "/.swarm-code/mcp-" ++ server ++ ".log"
}
```

## Proposed fix
```sw
fun mcp_log_path(server) {
    safe = string_replace(string_replace(server, "/", "_"), "\\", "_")
    getenv("HOME") ++ "/.swarm-code/mcp-" ++ safe ++ ".log"
}
```

## Risk
safe

## Confidence
high — `string_replace` is already used in this file (mcp_shq), and stripping `/` and `\` neutralizes directory traversal without changing the call signature or behavior for legitimate names.
