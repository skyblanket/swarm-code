# Patch 17 — llm.sw:858

## Problem
extract_content_impl does not guard against nil message before calling map_get(msg_obj, 'content'), causing a process crash on malformed API responses.

## Current code
```sw
            else {
                choice0 = hd(choices)
                msg_obj = map_get(choice0, 'message')
                map_get(msg_obj, 'content')
            }
```

## Proposed fix
```sw
            else {
                choice0 = hd(choices)
                msg_obj = map_get(choice0, 'message')
                if (msg_obj == nil) { nil }
                else { map_get(msg_obj, 'content') }
            }
```

safe

## Confidence
high — matches the existing nil-guard pattern already used in extract_reasoning on the next function
