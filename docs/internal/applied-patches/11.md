# Patch 11 — tools.sw:1190

## Problem
`do_file_watch` passes the raw `timeout_sec` argument directly into `with_timeout`, where it is interpolated into a shell command via `to_string(seconds)` without sanitization, allowing shell injection.

## Current code
```sw
        timeout = map_get(args, 'timeout_sec')
        timeout_n = if (timeout == nil) { 60 } else { timeout }
```

## Proposed fix
```sw
        timeout = map_get(args, 'timeout_sec')
        timeout_n = if (timeout == nil) { 60 } else { to_int(timeout) }
```

## Risk
safe

## Confidence
high — `to_int` (defined 200 lines below in the same file) calls `parse_int_safe`, which strips non-digits and returns a numeric value, eliminating the injection surface without changing behavior for valid inputs.
