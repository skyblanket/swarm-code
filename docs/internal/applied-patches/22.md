# Patch 22 — mcp.sw:145

## Problem
Boot deadline (45s) is shorter than the maximum time a boot worker can spend on initialize + tools/list (60s), so a slow-but-successful server is registered in ETS by the worker but never appended to `all_servers` because `mcp_collect_boots` has already timed out.

## Current code
```sw
# Overall boot budget — a worker self-limits each server to
# mcp_handshake_timeout_ms; this is that plus margin.
fun mcp_boot_deadline_ms() { mcp_handshake_timeout_ms() + 15000 }
```

## Proposed fix
```sw
# Overall boot budget — init + tools/list each take up to
# mcp_handshake_timeout_ms; this is twice that plus margin.
fun mcp_boot_deadline_ms() { mcp_handshake_timeout_ms() * 2 + 15000 }
```

## Risk
safe

## Confidence
high — single-line arithmetic change that directly closes the 45s/60s gap
