# Patch 01 — agent.sw:1566

## Problem
`collect_tool_result` blocks forever in `receive` with no timeout; a hung tool worker freezes the REPL.

## Current code
```sw
fun collect_tool_result(name, pending) {
    receive {
        {'tool_done', result} -> collect_tool_result(name, result)
        {'EXIT', _, ex_reason} ->
            if (pending == nil) { tool_crash_msg(name, ex_reason) }
            else { pending }
    }
}
```

## Proposed fix
```sw
fun collect_tool_result(name, pending) {
    receive {
        {'tool_done', result} -> collect_tool_result(name, result)
        {'EXIT', _, ex_reason} ->
            if (pending == nil) { tool_crash_msg(name, ex_reason) }
            else { pending }
        after 600000 {
            if (pending == nil) {
                "error: tool '" ++ to_string(name) ++ "' timed out (worker did not respond)"
            } else {
                pending
            }
        }
    }
}
```

## Risk
safe

## Confidence
high — `after` syntax is already used in the same file (`main_loop` at line 374) and in `mcp.sw`; 600000 ms matches the maximum bash tool timeout so legitimate long-running commands are not falsely cut off.
