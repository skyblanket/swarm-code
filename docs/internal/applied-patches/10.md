# Patch 10 — tools.sw:1149

## Problem
timeout_sec from user args is passed unsanitized into with_timeout, where to_string(seconds) is concatenated directly into a perl command string, enabling shell injection.

## Current code
```sw
        timeout = map_get(args, 'timeout_sec')
        timeout_n = if (timeout == nil) { 60 } else { timeout }
```

## Proposed fix
```sw
        timeout = map_get(args, 'timeout_sec')
        timeout_n = if (timeout == nil) { 60 } else { parse_int_safe(to_string(timeout), 60) }
```

## Risk
safe

## Confidence
high — parse_int_safe is already defined in the same file and used by resolve_bash_timeout_s for identical sanitization.
