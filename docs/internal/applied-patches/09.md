# Patch 09 — tools.sw:904

## Problem
`max_n` is injected into the shell command unsanitized, allowing shell metacharacters to be interpreted.

## Current code
```sw
        cmd = "SWC_WSQ=" ++ Util.shell_q(to_string(q)) ++
              " SWC_WSN=" ++ to_string(max_n) ++
              " python3 -c " ++ Util.shell_q(py_script)
```

## Proposed fix
```sw
        cmd = "SWC_WSQ=" ++ Util.shell_q(to_string(q)) ++
              " SWC_WSN=" ++ Util.shell_q(to_string(max_n)) ++
              " python3 -c " ++ Util.shell_q(py_script)
```

safe
