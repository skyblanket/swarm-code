# Patch 08 — tools.sw:1216

## Problem
shell_inner_quote only escapes `"`, not `$` or backticks, so a malicious path like `/tmp/$(cmd)` is executed by bash when spliced into the double-quoted assignment in do_file_watch.

## Current code
```sw
        inner =
            "p=" ++ shell_inner_quote(path) ++ "; " ++
            "initial=$(stat -f %m \"$p\" 2>/dev/null || echo missing); " ++
            "while true; do " ++
            "  current=$(stat -f %m \"$p\" 2>/dev/null || echo missing); " ++
            "  [ \"$current\" != \"$initial\" ] && echo \"changed: $initial -> $current\" && exit 0; " ++
            "  sleep 0.5; " ++
            "done"

# Escape for use INSIDE a double-quoted shell string (bash).
fun shell_inner_quote(s) {
    no_dq = string_replace(s, "\"", "\\\"")
    "\"" ++ no_dq ++ "\""
}
```

## Proposed fix
```sw
        inner =
            "p=" ++ Util.shell_q(path) ++ "; " ++
            "initial=$(stat -f %m \"$p\" 2>/dev/null || echo missing); " ++
            "while true; do " ++
            "  current=$(stat -f %m \"$p\" 2>/dev/null || echo missing); " ++
            "  [ \"$current\" != \"$initial\" ] && echo \"changed: $initial -> $current\" && exit 0; " ++
            "  sleep 0.5; " ++
            "done"
```

## Risk
safe

## Confidence
high — replaces a bespoke, incomplete escaper with the existing POSIX-safe `Util.shell_q` (single-quote wrap), eliminating `$`, backtick, and backslash injection in one line change.
